## Why

The backend already verifies AVIF format and size budgets for persisted media, but it still largely trusts compliant uploads as-is. That leaves a gap between media validation and true backend sanitization: malformed-but-decodable payloads, unexpected metadata, and non-canonical dimensions can still cross the server boundary and be stored as product media.

## What Changes

- Tighten the backend media contract for persisted artwork and avatar uploads so the server sanitizes accepted media instead of only validating MIME, size, and container signature.
- Define canonical backend behavior for decoding, normalizing, and re-encoding uploaded artwork and avatar images before storage.
- Enforce server-owned output invariants for persisted media, including canonical dimensions, stripped non-essential metadata, and a canonical AVIF output generated by the backend.
- Define failure behavior for uploads that cannot be decoded, normalized, or re-encoded safely.
- Add automated backend coverage for sanitized media persistence, invalid payload rejection, and storage-cleanup behavior when sanitization succeeds but persistence fails.

## Capabilities

### New Capabilities
- None.

### Modified Capabilities
- `artwork-publishing`: Artwork publish requirements expand from validating media format and size to sanitizing accepted uploads into canonical persisted artwork media before storage.
- `avatar-management`: Avatar upload requirements expand from validating media format and size to sanitizing accepted uploads into canonical persisted avatar media before storage.

## Impact

- Affected code: artwork and avatar upload services, shared media validation/sanitization utilities, storage integration code, route handlers that accept multipart media, and backend tests covering upload behavior.
- Affected systems: Supabase Storage object writes, backend media-processing pipeline, upload error handling, and environment/runtime dependencies required for image decoding and AVIF encoding.
- Dependencies: existing `artwork-publishing` and `avatar-management` capabilities, current AVIF and size-budget rules, and the backend trust-boundary principle that the server owns persisted media invariants.
- Follow-on work unlocked: stronger launch security posture for uploads, clearer moderation/media-processing hooks, and safer future expansion into derivative media workflows.
